## 📋 Step-by-Step Setup

### Step 1: Deploy Database Schema (1 minute)

The new tables have been added to `supabase/schema.sql`:
- `item_costs` - Track unit cost for each menu item
- `financial_expenses` - Track operating expenses

**To deploy:**

1. Go to Supabase > SQL Editor
2. Create a new query
3. Run this SQL to add the new tables:

```sql
-- Add total_amount column to orders (for revenue tracking)
alter table if exists public.orders 
add column if not exists total_amount numeric(10, 2) default 0 check (total_amount >= 0);

-- Item Costs: Track cost of goods sold per menu item
create table if not exists public.item_costs (
  id bigint generated by default as identity primary key,
  menu_item_id bigint not null references public.menu_items(id) on delete restrict,
  cost_per_unit numeric(10, 2) not null check (cost_per_unit >= 0),
  notes text,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  unique(menu_item_id)
);

-- Financial Expenses: Operating costs, utilities, payroll, etc.
create table if not exists public.financial_expenses (
  id bigint generated by default as identity primary key,
  title text not null,
  amount numeric(10, 2) not null check (amount > 0),
  type text not null check (type in ('fixed', 'variable')),
  category text,
  description text,
  expense_date date not null default CURRENT_DATE,
  created_at timestamptz not null default now()
);

-- Create indexes for performance
create index if not exists idx_item_costs_menu_item_id on public.item_costs(menu_item_id);
create index if not exists idx_financial_expenses_date on public.financial_expenses(expense_date desc);
create index if not exists idx_financial_expenses_type on public.financial_expenses(type);
create index if not exists idx_orders_total_amount on public.orders(total_amount);

-- Enable RLS on new financial tables
alter table public.item_costs enable row level security;
alter table public.financial_expenses enable row level security;

-- RLS Policies for item_costs
drop policy if exists "public read item_costs" on public.item_costs;
create policy "public read item_costs" on public.item_costs 
  for select to anon, authenticated using (true);

drop policy if exists "authenticated manage item_costs" on public.item_costs;
create policy "authenticated manage item_costs" on public.item_costs 
  for all to authenticated using (true) with check (true);

-- RLS Policies for financial_expenses
drop policy if exists "public read financial_expenses" on public.financial_expenses;
create policy "public read financial_expenses" on public.financial_expenses 
  for select to anon, authenticated using (true);

drop policy if exists "authenticated manage financial_expenses" on public.financial_expenses;
create policy "authenticated manage financial_expenses" on public.financial_expenses 
  for all to authenticated using (true) with check (true);

-- Publish new tables for real-time
do $$
begin
  if not exists (
    select 1 from pg_publication_tables
    where pubname = 'supabase_realtime' and schemaname = 'public' and tablename = 'item_costs'
  ) then
    alter publication supabase_realtime add table public.item_costs;
  end if;

  if not exists (
    select 1 from pg_publication_tables
    where pubname = 'supabase_realtime' and schemaname = 'public' and tablename = 'financial_expenses'
  ) then
    alter publication supabase_realtime add table public.financial_expenses;
  end if;
end $$;
```

## 📊 Financial Formulas

### Gross Profit
```
Gross Profit = Revenue - COGS
Gross Margin % = (Gross Profit / Revenue) × 100
```

### Cost of Goods Sold (COGS)
```
COGS = Σ(Order Item Quantity × Item Cost)
For each item in orders during period
```

### Net Profit
```
Net Profit = Gross Profit - Operating Expenses
Net Margin % = (Net Profit / Revenue) × 100
```

### Item Profitability
```
Item Profit = (Sell Price × Qty) - (Cost × Qty)
Item Margin % = (Item Profit / Item Revenue) × 100
```

## ⚙️ System Architecture

### Data Flow

```

        ↓
Customers specify items (existing)
        ↓

        ↓
```

## 📞 Troubleshooting

### "COGS shows 0"
→ **Solution**: Set item costs. Go to Finance Tab → Cost Management

### "No data in profitability table"
→ **Solution**: 
1. Make sure you have completed orders
2. Set item costs for those menu items
3. Expenses must be in correct period

### "Margins are not calculating"
→ **Solution**: Verify:
1. Item cost is set
2. Sell price is correct
3. Check that margin = (price - cost) / price

### "Financial data not updating"
→ **Solution**:
1. Wait a moment (real-time sync takes 1-2 seconds)
2. Refresh browser
3. Check browser console for errors
